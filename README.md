## 기본 개념

| 용어 | 설명 | 비유 |
|------|------|------|
| **Repository (레포지터리)** | 프로젝트 파일과 변경 이력을 저장하는 공간 | 📁 프로젝트 폴더 |
| **Commit (커밋)** | 변경 사항을 저장하는 행위 | 💾 저장 버튼 누르기 |
| **Branch (브랜치)** | 독립적으로 작업할 수 있는 평행 작업 공간 | 🌿 나뭇가지 |
| **Merge (머지)** | 브랜치를 다른 브랜치에 합치는 것 | 🔀 두 가지를 하나로 합치기 |
| **Push (푸시)** | 로컬 변경 사항을 GitHub에 올리는 것 | ⬆️ 업로드 |
| **Pull (풀)** | GitHub의 변경 사항을 로컬로 가져오는 것 | ⬇️ 다운로드 |
| **Clone (클론)** | 원격 레포지터리를 내 컴퓨터에 복사하는 것 | 📋 복사하기 |
| **Fork (포크)** | 다른 사람의 레포지터리를 내 계정으로 복사하는 것 | 🍴 복제하기 |
| **Pull Request (PR)** | 변경 사항을 합쳐달라고 요청하는 것 | 📨 합병 요청서 |
| **Issue (이슈)** | 버그나 기능 요청 등을 기록하는 곳 | 📝 할 일 목록 |

## 핵심 명령어

### 🔧 설정 (처음 한 번만)

| 명령어 | 설명 |
|--------|------|
| `git config --global user.name "이름"` | 사용자 이름 설정 |
| `git config --global user.email "이메일"` | 사용자 이메일 설정 |
| `git config --list` | 현재 설정 확인 |

### 📁 레포지터리 시작하기

| 명령어 | 설명 |
|--------|------|
| `git init` | 현재 폴더를 Git 레포지터리로 초기화 |
| `git clone [URL]` | 원격 레포지터리를 로컬로 복사 |

### 📸 변경사항 저장하기

| 명령어 | 설명 |
|--------|------|
| `git status` | 현재 변경 상태 확인 |
| `git add [파일명]` | 특정 파일을 스테이징 영역에 추가 |
| `git add .` | 모든 변경 파일을 스테이징 영역에 추가 |
| `git commit -m "메시지"` | 스테이징된 파일을 커밋 |
| `git log` | 커밋 이력 확인 |
| `git diff` | 변경된 내용 비교 |

### ☁️ 원격 레포지터리 (GitHub)

| 명령어 | 설명 |
|--------|------|
| `git remote add origin [URL]` | 원격 저장소 연결 |
| `git push origin main` | main 브랜치를 GitHub에 업로드 |
| `git pull origin main` | GitHub에서 최신 내용 가져오기 |
| `git remote -v` | 연결된 원격 저장소 확인 |

### 🌿 브랜치

| 명령어 | 설명 |
|--------|------|
| `git branch` | 브랜치 목록 확인 |
| `git branch [브랜치명]` | 새 브랜치 생성 |
| `git checkout [브랜치명]` | 해당 브랜치로 이동 |
| `git checkout -b [브랜치명]` | 브랜치 생성 + 이동 (한 번에) |
| `git merge [브랜치명]` | 현재 브랜치에 해당 브랜치를 합치기 |
| `git branch -d [브랜치명]` | 브랜치 삭제 |

## 작업 흐름

### 🚀 새 프로젝트 시작할 때

```
1. GitHub에서 새 레포지터리 생성
2. git clone [URL]
3. 파일 작업
4. git add .
5. git commit -m "첫 커밋"
6. git push origin main
```

### 🔄 일반적인 작업 흐름

```
1. git pull origin main        ← 최신 코드 받기
2. git checkout -b 새기능      ← 새 브랜치 만들기
3. 파일 수정 작업
4. git add .
5. git commit -m "기능 추가"
6. git push origin 새기능      ← GitHub에 올리기
7. GitHub에서 Pull Request 생성
```

## Markdown 문법

> README.md 파일을 예쁘게 꾸밀 때 사용하는 문법이에요!

| 문법 | 코드 | 결과 |
|------|------|------|
| **제목 1** | `# 제목` | 가장 큰 제목 |
| **제목 2** | `## 제목` | 두 번째 크기 제목 |
| **제목 3** | `### 제목` | 세 번째 크기 제목 |
| **굵게** | `**텍스트**` | **텍스트** |
| **기울임** | `*텍스트*` | *텍스트* |
| **취소선** | `~~텍스트~~` | ~~텍스트~~ |
| **인라인 코드** | `` `코드` `` | `코드` |
| **링크** | `[표시](URL)` | 클릭 가능한 링크 |
| **이미지** | `![설명](이미지URL)` | 이미지 표시 |
| **수평선** | `---` | 구분선 |
| **인용** | `> 텍스트` | 들여쓰기 인용 |
| **순서 없는 목록** | `- 항목` | • 항목 |
| **순서 있는 목록** | `1. 항목` | 1. 항목 |
| **체크박스** | `- [ ] 항목` | ☐ 항목 |
| **완료 체크박스** | `- [x] 항목` | ☑ 항목 |
| **테이블** | `\| 열1 \| 열2 \|` | 표 형식 |

## 자주 쓰는 Git 상황별 명령어

| 상황 | 명령어 | 설명 |
|------|--------|------|
| 실수한 커밋 메시지 수정 | `git commit --amend -m "새 메시지"` | 직전 커밋 메시지 변경 |
| 파일 변경 취소 (커밋 전) | `git checkout -- [파일명]` | 수정 사항 되돌리기 |
| 스테이징 취소 | `git reset HEAD [파일명]` | add 취소하기 |
| 직전 커밋 취소 | `git reset --soft HEAD~1` | 커밋만 취소 (파일은 유지) |
| 특정 커밋으로 되돌리기 | `git revert [커밋ID]` | 안전하게 되돌리기 |
| 변경 내용 임시 저장 | `git stash` | 작업 내용 잠깐 보관 |
| 임시 저장 복원 | `git stash pop` | 보관한 내용 꺼내기 |
| 파일 삭제 후 추적 제거 | `git rm [파일명]` | Git에서도 파일 삭제 |
| .gitignore 적용 새로고침 | `git rm -r --cached .` | 캐시 초기화 후 재추가 |

## 📝 좋은 커밋 메시지 작성법

| 유형 | 예시 | 설명 |
|------|------|------|
| `feat` | `feat: 로그인 기능 추가` | 새로운 기능 |
| `fix` | `fix: 버튼 클릭 오류 수정` | 버그 수정 |
| `docs` | `docs: README 업데이트` | 문서 수정 |
| `style` | `style: 들여쓰기 정리` | 코드 포맷 변경 |
| `refactor` | `refactor: 함수 구조 개선` | 코드 리팩토링 |
| `test` | `test: 유닛 테스트 추가` | 테스트 추가 |
| `chore` | `chore: 패키지 버전 업데이트` | 기타 변경 |

## ⚠️ .gitignore 란?

Git이 **무시할 파일 목록**을 적어두는 파일이에요.  
비밀번호, API 키, 자동 생성 파일 등은 GitHub에 올리면 안 돼요!

```
# .gitignore 예시
.env              # 환경변수 (비밀번호, API키)
node_modules/     # 패키지 폴더 (용량 큼)
.DS_Store         # Mac 시스템 파일
*.log             # 로그 파일
```
